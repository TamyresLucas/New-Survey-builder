## [1.0.0] - 2026-01-20

### Added

- Configured Tailwind CSS to scan local design system package and created a debug component for integration verification.

### Changed

- Updated Command component to use 'bg-primary/10' for selected state, matching DropdownMenu tokens
- Updated Badge component to use normal font weight for 'outline', 'destructive', 'success', 'warning', and 'secondary' variants (default remains bold)
- Added 'ghost-primary' button variant with primary color text at 100% opacity by default, full primary background on hover with primary-foreground text for WCAG compliant contrast
- Standardized form components hover state (Input, Select, Textarea, InputOTP): border changes to primary 100%, background changes to primary 20% for consistent interactive feedback
- Refactored all component borders (light and dark mode) to use 'Primary 20%' (border-primary/20) by updating global CSS tokens, removing black/dark borders completely
- Updated Table and DataGrid row hover states to maintain standard foreground text color (referencing standard foreground token) instead of accent foreground
- Updated Table and DataGrid selected row text color to use 'Primary 100%' and added 2px left border (via box-shadow) while maintaining default background
- Implemented 'TableRowActions' component to standardize row actions and updated 'Table' and 'DataGrid' stories to use this new pattern
- Token synchronization: Survey Builder now imports Design System CSS for unified tokens
- Refined Alert, Badge, and Toast styles: Default uses Soft Primary (10% bg, 40% border), 'info' uses primary border, and all semantic borders set to 40% opacity
- Extended 'Solid Tinted' styling logic to Card component, adding semantic variants (destructive, success, warning, info) with consistent 40% border opacity
- Updated form elements (Input, Textarea, Select, Checkbox) to implement 'destructive' state border guidelines (40% opacity) via aria-invalid attribute
- Standardized default form element borders to use Primary color with 40% opacity. Checkbox/Radio follow text color. Badge 'default' updated to Solid Primary style (no border, 100% bg). Badge 'outline' and Color Picker overlay use Primary 40% border
- Implemented standardized 'Disabled' state across form components (Button, Input, Textarea, Select, Checkbox, RadioGroup, Switch) using 'disabled:text-primary/40' for consistent branded appearance
- Enforced brand consistency on 'Disabled' states across all Shadcn UI components (including Toggle, Slider, Tabs, Select Item, Dropdown/Context/Menubar, Command, and Calendar) using 'text-primary/40' via native disabled and Radix data (data-[disabled]) attributes
- Updated 'Disabled' background color for all filled components (Button, Checkbox, Switch, Toggle, Slider) to use 'Primary 10%' (mixed with background for capacity) opacity token to ensure consistent branding even in disabled states
- Updated UserMessage component to use branded Primary 10% background and Primary 40% border, matching the system's soft aesthetic
- Updated Button disabled text color to 'text-primary/40' for all variants (default, destructive, success, outline, secondary, ghost, link) to ensure consistent branded disabled state
- Updated Button outline variant border to 'border-primary/40' to align with the 'solid tinted' aesthetic for bordered components
- Updated Button secondary variant background to 'Primary 10%' (mixed with background for capacity) to ensure a consistent soft tinted appearance
- Updated Button 'outline', 'secondary', and 'ghost' variants hover state to use 'Primary 20%' (mixed with 80% background) for a consistent and branded interaction feedback
- Updated DiagramCanvas background to use 'Primary 10%' (mixed with background) opaque token for consistent branding
- Updated DiagramCanvas edges to use 'text-foreground' color by default, switching to 'primary' color when selected
- Updated Composer component to use 'border-primary/40' for the input field, aligning with the standardized form element styling
- Updated AssistantSidebar components (sidebar border, header border, composer divider) to use 'border-primary/20' for a consistent subtle branded separation
- Updated AccordionItem border to use 'border-primary/40' to ensure consistent branding across all bordered components
- Updated AvatarFallback to use 'Primary 10%' (mixed with background) background and 'text-primary' to ensure consistent soft branding
- Updated 'muted' generic color token to use 'Primary 10%' (mixed with background) to ensure consistent subtle branding across all muted state components
- Updated Card default border to use 'border-primary/20' to align with the standardized subtle border visibility
- Updated StatsCard to dynamically use 'Destructive' or 'Success' Card variants based on trend, ensuring exact semantic match with Alerts
- Updated Table header background to explicitly use 'Primary 10%' (mixed with background) for consistent branding
- Updated Table and DataGrid hover states to use 'Primary 20%' (mixed with background) for standardized interactive feedback
- Updated AssistantMessage bubble to use 'border-primary/20' to align with the subtle branded border standard
- Updated Carousel example cards (Statements and Survey Question Types) to use 'border-primary/40' to meet specific visibility requirements
- Updated Progress bar background to use 'Primary 20%' (mixed with background) to ensure consistent soft branding
- Updated Carousel indicators (dots) background to use 'Primary 20%' (mixed with background) for standardized visibility
- Updated TableFooter border to use 'border-primary/40' to meet standardization requirements
- Updated Table and DataGrid row borders to use 'Primary 20%' (mixed with background) for consistent grid visibility
- Fixed DataGrid border visibility by enforcing 'Primary 20%' mix with !important rule and updating internal grid variables
- Updated Table and DataGrid selected row backgrounds to use 'Primary 20%' (mixed with background) for consistent state indication
- Updated DropdownMenu border to use 'border-primary/20' and Separators to use 'primary/20' for hierarchical clarity
- Updated HoverCard border to use 'border-primary/20' to align with soft aesthetic standards
- Updated Menubar border to use 'border-primary/40', with sub-menus using 'primary/20' and separators 'primary/40'
- Updated Menubar hover and focus states to use 'Primary 20%' (mixed with background) for consistent interaction feedback
- Updated Popover border to use 'border-primary/20' to align with soft aesthetic standards
- Updated Tooltip border to use 'border-primary/20' for consistent micro-interaction styling
- Updated Breadcrumb WithBadge story to use standard 'Badge variant=secondary' component instead of inline styles
- Updated BreadcrumbLink to use 'hover:text-primary' and 'hover:underline' for distinct clickable feedback
- Updated NavigationMenu to use 'Primary 20%' background on hover/focus and 'border-primary/20' for dropdowns
- Updated Stepper inactive border to use 'border-primary/40' to maintain brand visibility in all states
- Updated Stepper content cards to use 'border-primary/20' for consistent container visualization
- Updated Tabs unread badge to use 'destructive' variant for clearer status indication
- Updated Tabs UnderlineStyle with 'border-primary/20' container, 'hover:text-primary' and 'data-[state=active]:text-primary' for labels
- Updated TabsTrigger with 'hover:text-primary' and 'data-[state=active]:hover:text-primary' for consistent hover feedback
- Updated VerticalTabs with 'w-full', 'rounded-none', border-right styling, and 'data-[state=active]:text-primary' for sidebar-style navigation
- Changed default Tabs styling to underline style with transparent background and border-primary indicators
